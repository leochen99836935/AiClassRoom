## 1. 背景

AIClassRoom 的第 0 号任务：把空仓库变成一个可 `pnpm install` 的 pnpm monorepo 骨架。在此之后，所有后续任务（TS、lint、types、Next.js 等）都站在这个地基上。

技术选型由 [docs/tech-stack.md § 1](../tech-stack.md) 确认：**pnpm 9+ workspaces**，不引入 Turborepo / Nx / Changesets（需要时另开 spec）。

**现状**：仓库里已有上一轮探索留下的 stale 脚手架文件。本 spec 的实现步骤会**审计并规范化**它们；凡与本 spec 不符的字段一律以本 spec 为准。

## 2. 用户故事

- 作为开发者，我 clone 仓库后执行 `pnpm install` 就得到完整工作区，立即可以开工。
- 作为贡献者，`apps/web` 用 `workspace:*` 引用 `@aiclassroom/types`，我改 types 源码 web 立刻可见，不用发包。

## 3. 范围

### 3.1 In Scope

- 根文件：`package.json` / `pnpm-workspace.yaml` / `.gitignore` / `.npmrc` / `.nvmrc` / `README.md`
- workspace 拓扑：`apps/*` + `packages/*`
- 子包占位：`apps/web` + `packages/{types,ui,prompts,agents,llm,tts,config}` 各自的 `package.json` + 空 `src/index.ts`
- 子包间依赖一律用 `workspace:*`
- Node `>=20.11.0`、`packageManager: "pnpm@9.12.0"` 硬固定

### 3.2 Out of Scope

- TypeScript 配置 → P0.A-2
- ESLint / Prettier 配置 → P0.A-3（但根 `devDependencies` 保留 `prettier` / `typescript`，因 A-2/A-3 立即要用）
- Vitest → P0.A-4
- `pnpm check` 聚合 + git hooks → P0.A-5
- 任何业务代码
- Next.js `dev` 脚本的真正可用 → P0.C-1（本 spec 允许 `pnpm dev` 失败）

## 5. 关键文件内容

### 5.1 根 `package.json`

```json
{
  "name": "aiclassroom",
  "private": true,
  "version": "0.0.0",
  "description": "Enterprise AI interactive classroom — doc-driven / vibe coding",
  "packageManager": "pnpm@9.12.0",
  "engines": { "node": ">=20.11.0" },
  "scripts": {
    "dev": "pnpm --filter web dev",
    "build": "pnpm -r build",
    "lint": "pnpm -r lint",
    "typecheck": "pnpm -r typecheck",
    "test": "pnpm -r test",
    "check": "pnpm lint && pnpm typecheck && pnpm test"
  },
  "devDependencies": {
    "prettier": "^3.3.3",
    "prettier-plugin-tailwindcss": "^0.6.8",
    "typescript": "^5.6.0"
  }
}
```

### 5.2 子包 `package.json`（模板）

```json
{
  "name": "@aiclassroom/<name>",
  "version": "0.0.0",
  "private": true,
  "main": "./src/index.ts",
  "types": "./src/index.ts",
  "scripts": {
    "lint": "echo skip",
    "typecheck": "echo skip",
    "test": "echo skip",
    "build": "echo skip"
  }
}
```

`apps/web/package.json` 同上但 `name: "web"` 且无 `main`/`types`。

**约定**：

- 所有子包脚本以 `echo skip` 占位，让 `pnpm -r <cmd>` 始终 exit 0；后续 spec 逐步替换。
- TypeScript 不下沉到子包，由根统一提升。
- `packages/config` 无 `src/`，只存共享配置文件（A-2 写入 `tsconfig.base.json`）。

### 5.3 `.npmrc`

```
auto-install-peers=true
strict-peer-dependencies=false
save-workspace-protocol=rolling
link-workspace-packages=true
shared-workspace-lockfile=true
```

### 5.4 `.nvmrc`

```
20.11.0
```

### 5.5 `pnpm-workspace.yaml`

```yaml
packages:
  - "apps/*"
  - "packages/*"
```

### 5.6 `src/index.ts` 占位

每个有 `src/` 的子包：

```ts
export {};
```

## 6. 实现步骤

```
1. 审计现有文件：对照 § 4 拓扑列出差异
2. 按 § 5 覆写/新建所有文件
3. 清理 stale：packages/*/src/index.ts 统一为 `export {};`
4. 删除旧 pnpm-lock.yaml（若与新配置冲突）
5. pnpm install → 生成新 lockfile
6. 跑 § 8 全部 AC
7. git add 所有变更（不 commit，由用户决定）
```

## 7. 失败模式

| 场景                                               | 行为                                           |
| -------------------------------------------------- | ---------------------------------------------- |
| Node < 20.11                                       | pnpm 报 engines 警告/错误，README 提示升级     |
| 误用 npm / yarn                                    | Corepack 警告；README 明确只用 pnpm            |
| `apps/web` 的 `name` 与 `pnpm --filter web` 不匹配 | pnpm 报 "No projects matched" → 必须是 `"web"` |

## 8. 验收标准

- **AC-1**: `pnpm install` exit 0，生成 `pnpm-lock.yaml`
- **AC-2**: `pnpm ls --depth -1` 列出 9 个条目：`aiclassroom` 根 + `web` + `@aiclassroom/{types,ui,prompts,agents,llm,tts,config}`
- **AC-3**: `pnpm -r lint` / `typecheck` / `test` / `build` 全部 exit 0（打印 skip）
- **AC-4**: `pnpm --filter @aiclassroom/types typecheck` 能定位到目标包
- **AC-5**: 根 `package.json.packageManager === "pnpm@9.12.0"`、`engines.node === ">=20.11.0"`
- **AC-6**: `.nvmrc` / `.npmrc` 存在且内容与 § 5.3 / § 5.4 一致
- **AC-7**: `apps/web/package.json.name === "web"`；所有 `packages/<x>/package.json.name === "@aiclassroom/<x>"`

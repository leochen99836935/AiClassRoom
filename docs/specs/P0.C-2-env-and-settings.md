## 1. 背景与动机

所有环境变量需要用 zod 做运行时校验，在应用启动时验证必要配置。同时需要一个 settings Zustand store 存放客户端可切换的配置（主题、语言等），为后续 P1 的设置弹窗做地基。

## 2. 用户故事

- 作为开发者，我希望缺 env 时启动即报错，以便快速定位配置问题。
- 作为用户，我希望能切换主题和语言，且选择在刷新后保留。

## 3. 范围

### 3.1 In Scope

- `apps/web/src/env.ts`：用 zod 声明并校验所有 env（参照 `docs/tech-stack.md` §9）
- `apps/web/src/stores/settings-store.ts`：Zustand + Immer，管理 theme / locale 偏好
- 服务端 env（API Key 等）仅在 server 使用，不暴露到客户端
- 客户端 env（`NEXT_PUBLIC_*`）通过 zod 校验

### 3.2 Out of Scope

- Dexie 持久化（P0.C-3）
- 模型/语音配置 UI（P1.A-4/P1.A-5）
- API Key 加密存储（P1）

## 4. 数据结构

### 4.1 Server Env Schema

```typescript
const serverEnvSchema = z.object({
  ANTHROPIC_API_KEY: z.string().optional(),
  OPENAI_API_KEY: z.string().optional(),
  GOOGLE_GENERATIVE_AI_API_KEY: z.string().optional(),
  DEEPSEEK_API_KEY: z.string().optional(),
  LOCAL_LLM_BASE_URL: z.string().url().optional(),
  LOCAL_LLM_API_KEY: z.string().optional(),
  OPENAI_TTS_KEY: z.string().optional(),
  MINIMAX_API_KEY: z.string().optional(),
});
```

### 4.2 Client Env Schema

```typescript
const clientEnvSchema = z.object({
  NEXT_PUBLIC_APP_NAME: z.string().default("AIClassRoom"),
  NEXT_PUBLIC_DEFAULT_LOCALE: z.enum(["zh-CN", "en-US"]).default("zh-CN"),
});
```

### 4.3 Settings Store Shape

```typescript
interface SettingsState {
  theme: "light" | "dark" | "system";
  locale: "zh-CN" | "en-US";
}

interface SettingsActions {
  setTheme: (theme: SettingsState["theme"]) => void;
  setLocale: (locale: SettingsState["locale"]) => void;
}
```

## 6. 验收标准

- **AC-1**: `serverEnv` 在服务端可用，类型安全
- **AC-2**: `clientEnv` 在客户端/服务端均可用
- **AC-3**: settings store 能读写 theme 和 locale，默认值正确
- **AC-4**: `pnpm check` 全绿

## 1. 背景与动机

首页是产品的第一印象。用户在这里输入课程主题、配置设置，然后提交进入生成流程。参照 `01-首页.png` 设计稿。

## 2. 用户故事

- 作为用户，我打开首页，看到居中大 Logo + slogan + 聊天式输入框，直觉地知道要在这里输入课程主题。
- 作为用户，我能在右上角看到语言、主题、设置的入口。

## 3. 范围

### 3.1 In Scope

- 居中大 Logo + slogan（"Generative Learning in Multi-Agent Interactive Classroom"）
- 聊天式输入框区域（底部居中，含示例头像 + 工具栏占位）
- 右上角工具栏占位（语言、主题、设置图标按钮）
- 提交事件触发（console.log 占位，Phase 1 后续任务接入 store）
- 响应式设计：320 ~ 1920px

### 3.2 Out of Scope

- 语言切换功能（P1.A-2）
- 主题切换功能（P1.A-3）
- 设置弹窗（P1.A-4 ~ P1.A-6）
- 文件上传（Phase 2）
- 语音输入（Phase 2）

## 4. UI / 交互

### 4.1 布局

```
┌────────────────────────────────────────┐
│                    [lang] [theme] [⚙]  │  ← 右上角工具栏
│                                        │
│            🎓 AIClassRoom              │  ← 居中 Logo
│   Generative Learning in Multi-Agent   │
│        Interactive Classroom           │
│                                        │
│ ┌────────────────────────────────────┐ │
│ │ 👤 角色  [生成课堂 ▸▸] [头像组]    │ │  ← 聊天式输入区
│ │                                    │ │
│ │ 输入文本内容...                     │ │
│ │                                    │ │
│ │ [📎] [🖼] [🎤] [🔍] [💡]  [发送▸] │ │  ← 工具栏 + 发送
│ └────────────────────────────────────┘ │
│         AIClassRoom · Open Source      │  ← footer
└────────────────────────────────────────┘
```

### 4.2 交互

- 输入框支持多行文本（textarea auto-resize）
- Enter 发送，Shift+Enter 换行
- 发送按钮在有内容时高亮
- 工具栏按钮 Phase 1 只做 UI 占位，点击无效果

## 1. 背景与动机

客户端需要 IndexedDB 本地持久化用于：草稿 Lesson、会话历史、API Key 加密存储、Trace 记录。Phase 0 只建表结构 + 升级脚手架，不做加密逻辑。

## 2. 范围

### 2.1 In Scope

- `apps/web/src/lib/db.ts`：Dexie 实例 + v1 schema
- 4 张表：`drafts`、`lessons`、`apiKeys`、`traces`
- 索引定义（按查询场景）

### 2.2 Out of Scope

- 加密存储逻辑（P1）
- 数据迁移（v2+）
- 服务端持久化（Phase 3）

## 3. Schema

### v1 tables

```typescript
drafts: "id, updatedAt"; // 草稿 Lesson
lessons: "id, status, createdAt"; // 完成的 Lesson
apiKeys: "provider"; // provider 做主键
traces: "++id, lessonId, agent, startedAt"; // auto-increment id
```

## 4. 验收标准

- **AC-1**: `db.ts` 导出 Dexie 实例，`pnpm typecheck` 全绿
- **AC-2**: 打开页面后 IndexedDB 中可见空表
- **AC-3**: `pnpm check` 全绿

## 5. 测试计划

- 由于 Dexie 依赖浏览器 IndexedDB，Phase 0 不写单元测试；AC-2 通过手动验证

## 1. 背景与动机

CLAUDE.md 规定"所有 LLM prompt 放 `packages/prompts/`，禁止在业务代码里内联长 prompt"。当前 `packages/prompts/` 只有占位文件。

本 spec 目标：

1. 建立 **prompt 文件约定**：每个 prompt 文件必须导出 `version` 和 `template`
2. 写一份 **约定说明文档** `_convention.md`
3. 写第一条 **hello prompt**（P0.C-4 Hello Claude 烟囱会用到）
4. 提供 **类型定义** 让后续 prompt 文件有统一的结构

## 5. 关键设计

### 5.1 `PromptDef<TVariables>` 类型

```ts
export interface PromptDef<TVariables extends Record<string, string> = Record<string, string>> {
  /** 语义化版本号，如 "1.0.0" */
  version: string;
  /** prompt 名称，如 "hello" */
  name: string;
  /** prompt 模板字符串，用 {{variable}} 占位 */
  template: string;
  /** 渲染模板：将变量替换进 template */
  render: (vars: TVariables) => string;
}
```

### 5.2 hello prompt

```ts
export const helloPrompt: PromptDef<{ topic: string }> = {
  version: "1.0.0",
  name: "hello",
  template:
    "You are a helpful AI teaching assistant. The user wants to learn about {{topic}}. Say hello and briefly introduce the topic in 2-3 sentences.",
  render: (vars) => template.replace("{{topic}}", vars.topic),
};
```

### 5.3 约定（`_convention.md` 核心内容）

1. 每个 prompt 一个 `.ts` 文件，文件名 = prompt 名（kebab-case）
2. 每个文件导出一个 `PromptDef` 常量，命名为 `xxxPrompt`
3. `version` 遵循语义化版本：改措辞 patch，改结构 minor，改意图 major
4. `template` 中变量用 `{{variable}}` 格式
5. `render` 是纯函数，只做字符串替换
6. 禁止在 prompt 文件中 import 业务逻辑或副作用

## 6. 验收标准

- **AC-1**: **Given** 所有文件就位，**When** 执行 `pnpm check`，**Then** exit 0
- **AC-2**: **Given** `helloPrompt`，**When** 调用 `helloPrompt.render({ topic: "AI" })`，**Then** 返回的字符串包含 "AI" 且不包含 `{{topic}}`
- **AC-3**: **Given** `helloPrompt`，**When** 读取 `version`，**Then** 为合法的语义化版本字符串
- **AC-4**: **Given** `_convention.md`，**When** 阅读内容，**Then** 包含文件命名、导出格式、版本管理三条规则

## 7. 测试计划

- `render` 替换变量正确
- `version` 符合 semver 格式
- `template` 包含 `{{topic}}` 占位符

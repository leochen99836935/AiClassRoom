## 1. 背景与动机

第一次跑通 Vercel AI SDK + trace 的完整链路：用户输入 → Route Handler → LLM 流式生成 → 页面渲染 + trace 记录。验证 `packages/llm` 封装在真实 Next.js 环境下可用。

## 2. 范围

### 2.1 In Scope

- `apps/web/src/app/api/hello/route.ts`：POST handler，接收 messages，用 `@aiclassroom/llm` 的 `stream()` 流式返回
- `apps/web/src/app/hello/page.tsx`：客户端页面，用 `useChat` 展示对话 + 简单 trace 面板

### 2.2 Out of Scope

- 完整 trace panel UI（P1.D-1）
- 多模型切换（P1.A-4）
- 错误重试（P1.D-2）

## 3. API

| 方法 | 路径         | 请求                      | 响应                     |
| ---- | ------------ | ------------------------- | ------------------------ |
| POST | `/api/hello` | `{ messages: Message[] }` | SSE stream (AI SDK 协议) |

## 4. 验收标准

- **AC-1**: Given 设置了 `ANTHROPIC_API_KEY`，When 在 `/hello` 页面输入问题并提交，Then 页面流式显示 Claude 回复
- **AC-2**: Route handler 使用 `packages/llm` 的 `resolveModel` + `streamText`
- **AC-3**: `pnpm check` 全绿

## 1. 背景与动机

tech-stack.md 指定 Radix UI + shadcn/ui 作为通用组件方案。本 spec 在 `packages/ui` 中建立基础组件集，供 `apps/web`（P0.C-1）引用。

## 2. 范围

### 2.1 In Scope

- `cn()` 工具函数（clsx + tailwind-merge）
- 6 个基础组件：Button、Input、Dialog、Popover、Tabs、Toast
- 每个组件遵循 shadcn/ui 惯例：forwardRef、className 合并、Radix primitive 封装
- 组件从 `@aiclassroom/ui` 统一导出

### 2.2 Out of Scope

- Tailwind 配置（在 P0.C-1 的 apps/web 中配置）
- 组件视觉测试 / Storybook
- 业务组件（角色卡、播放控制等）

## 3. 依赖

新增到 `packages/ui/package.json`：

- `react`, `react-dom` (peerDependencies)
- `@radix-ui/react-dialog`
- `@radix-ui/react-popover`
- `@radix-ui/react-tabs`
- `@radix-ui/react-toast`
- `@radix-ui/react-slot`
- `class-variance-authority`
- `clsx`
- `tailwind-merge`
- `lucide-react`


## 5. 验收标准

- **AC-1**: `pnpm check` 全绿
- **AC-2**: `import { Button, Input, Dialog, Popover, Tabs, Toast } from "@aiclassroom/ui"` 可解析
- **AC-3**: 所有组件接受 `className` prop 并通过 `cn()` 合并

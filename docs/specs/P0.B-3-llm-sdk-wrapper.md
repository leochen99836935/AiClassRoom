## 1. 背景与动机

CLAUDE.md + tech-stack.md 要求"所有 LLM 调用必须走 `packages/llm`，禁止业务代码直接 import `@ai-sdk/*`"。本 spec 建立这个薄封装层。

核心职责：

1. `resolveModel(cfg)` — 根据配置构造 Vercel AI SDK 的 `LanguageModel`
2. `generateStructured(opts)` — 调用 `generateObject` 返回结构化 JSON
3. `stream(opts)` — 调用 `streamText` 返回流式文本
4. 每次调用产出 `TraceRecord`（来自 `@aiclassroom/types`）

## 2. 范围

### 2.1 In Scope

- `packages/llm/src/config.ts` — `ModelConfig` 类型
- `packages/llm/src/resolve-model.ts` — `resolveModel` 函数
- `packages/llm/src/generate-structured.ts` — `generateStructured` 函数
- `packages/llm/src/stream.ts` — `stream` 函数
- `packages/llm/src/trace.ts` — `createTrace` / `finalizeTrace` helper
- 重写 `index.ts` 和 `index.test.ts`

### 2.2 Out of Scope

- 实际调用 LLM（测试用 mock）
- LangGraph 集成（Phase 1）
- 重试/超时逻辑（后续 spec 增强）
- Key 加密存储（P0.C-3 Dexie）

## 3. 依赖

新增到 `packages/llm/package.json`：

- `ai` (Vercel AI SDK)
- `@ai-sdk/anthropic`
- `@ai-sdk/openai`
- `@ai-sdk/google`
- `@ai-sdk/deepseek`
- `@ai-sdk/openai-compatible`
- `zod`（generateObject 需要 schema）
- `@aiclassroom/types` (workspace)

## 4. 关键类型

```ts
export interface ModelConfig {
  provider: "anthropic" | "openai" | "google" | "deepseek" | "openai-compatible";
  model: string;
  apiKey?: string;
  baseUrl?: string; // openai-compatible 用
}

  grade: { provider: "anthropic", model: "claude-haiku-4-5-20251001" },
```

## 5. 验收标准

- **AC-1**: `pnpm check` 全绿
- **AC-2**: `resolveModel({ provider: "anthropic", model: "claude-sonnet-4-6" })` 返回有效 `LanguageModel`
- **AC-3**: `generateStructured` / `stream` 的类型签名正确，接受 zod schema / messages
- **AC-4**: `createTrace` / `finalizeTrace` 产出符合 `TraceRecord` 结构的对象

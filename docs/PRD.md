# AIClassRoom — 产品需求文档 (PRD)

> 本文档是产品真相来源。所有 UI、交互、文案都以本文档为准；与线框图冲突时，以本文档文字描述为准，并回头修正线框。

## 1. 产品定位

**AIClassRoom** 是一个企业级 AI 互动课堂平台：用户输入一个主题或上传资料，系统一键生成一堂由 AI 老师和 AI 同学共同参与的沉浸式互动课程，涵盖讲解、提问、板书、讨论、测验等环节。

- **目标用户**：企业内训、职业教育、K12 到大学的课堂辅助、个人终身学习者。
- **核心价值**：把"被动看文档/视频"升级为"沉浸式多智能体课堂"。
- **与 OpenMAIC 的差异**：
  - 更强的企业级能力：权限、审计、数据持久化、SSO（见阶段 4）。
  - 文档驱动开发（vibe coding）——所有功能必须先有 spec 再落地。
  - 默认 Claude（Sonnet 4.6 / Opus 4.6）优先，但保留多 provider。

## 2. 核心用户故事

1. **作为培训负责人**，我希望输入"新员工信息安全培训"，5 分钟内得到一节可播放的 AI 课程，能导出 PPT/HTML 发给员工。
2. **作为学习者**，我希望进入课堂后看到 AI 老师讲课、AI 同学提问，并能随时举手提问，得到针对我水平的解答。
3. **作为课程开发者**，我希望对生成好的课程进行二次编辑：调整大纲、换角色、改台词、插入自定义 slide。
4. **作为管理员**，我希望看到团队的学习记录、测验成绩、课堂时长（阶段 4）。

## 3. 页面与功能清单

> 括号内是 `.claude/images/` 中的参考图。

### 3.1 首页 / 课程生成入口 (`01-首页.png`)

- 居中 OpenMAIC 风格 Logo + slogan
- 聊天式输入框（底部），占位文字为示例主题
- 左上角头像区（已登录时显示）
- 右上角：语言切换 (`01-首页-中译文切换.png`)、主题切换 (`01-首页-主题切换.png`)、设置 (`01-首页-设置.png`)
- 设置弹窗内含：
  - **模型配置** (`01-首页-配置模型.png`)：选择 provider + model + API Key
  - **语音配置** (`01-首页-语音配置.png`)：TTS/ASR provider 与音色
  - **角色选择** (`01-首页-角色选择.png`)：从角色库中勾选本次课堂参与的 5 个角色
- 输入框工具条：附件（上传 PDF/PPT/MD）、图片、语音输入、搜索增强 toggle、深度思考 toggle
- 提交后进入"生成流程"

### 3.2 角色确认页 (`02-课堂角色.png`, `02-生成课堂角色.png`)

- 顶部标题"你的课堂角色"
- 横向 5 张角色卡片：1 位老师 + 4 位同学
- **角色来源（Phase 1）**：
  - 用户在首页设置弹窗 (`01-首页-角色选择.png`) 里**手动勾选**过 → 使用用户选择
  - 用户未勾选 → 使用**默认角色组**（角色库 `packages/types/roles.ts` 或 `features/home/roles/role-library.ts` 中标记为 `default: true` 的一组）
  - **Phase 1 不做 AI 生成角色**；不做"按主题智能匹配"
- **Phase 2+ 扩展**：在角色确认页加"按主题重新生成一组"按钮，调用 `role_agent` 基于当前课程主题产生新人设
- 卡片内容：头像、姓名、角色标签（老师/学生）、一句话人设
- 底部"继续"按钮
- 加载态：角色卡骨架屏（只在首次从 Dexie / 角色库 hydrate 时短暂出现）

### 3.3 大纲生成页 (`03-生成课程大纲.png`)

- 中央大卡片：展示一本"书"的拟物图示 + "生成课程大纲"标题
- 副标题："正在构建学习路径..."
- 底部 AI 智能体工作中状态条 + "查看详情"按钮（点击展开 agent trace）

### 3.4 教学动作生成页 (`04-生成教学动作.png`)

- 同样居中卡片，图示为 action timeline
- 标题"生成教学动作"，副标题"正在编排讲解、聚焦和互动流程..."
- 底部同 3.3 的状态条

### 3.5 课堂主界面 (`05-课堂主界面.png`, `06-课堂左侧PPT页面.png`)

**布局**：

- 顶部：面包屑（当前场景）+ 课程标题 + 右上工具栏（语言、投屏、分享、下载、用户）
- **左列**（可折叠）：场景列表 / PPT 缩略图 + 当前教师头像
- **中央主区**（上）：当前 slide / 白板 / 互动组件
- **中央底部**：对话区（当前说话者的气泡 + 自动滚动字幕）
- **右列**（可切 Tab）：笔记 / 对话记录 / 任务 Todo
- **底部控制条**：上一步 / 播放 / 下一步 / 速度 / 循环 / 标注
- **右下**：同学头像列（可点击互动）+ 用户麦克风按钮

### 3.6 角色介绍浮层 (`07-课堂老师介绍.png`, `08-课堂学生介绍.png`)

- 点击左下/右下头像 → 弹出该角色的详细人设卡
- 内含：头像、姓名、性格、擅长领域、常用语风格

### 3.7 场景加载态 (`09-课堂生成PPT界面.png`)

- 进入未生成的场景时，主区显示环形 loading + "场景正在生成，请稍候..."
- 加载完成自动切入渲染

## 4. 核心交互流程

```
[首页输入主题]
    │
    ▼
[加载角色]  ──(并行)──  [加载用户素材]
  ↑ Phase 1: 从固定角色库取"默认组"或用户在首页勾选的 5 位
  ↑ Phase 2+: 可选 role_agent 按主题生成新角色
    │
    ▼
[确认角色] ── 用户点「继续」
    │
    ▼
[生成大纲] ── LangGraph: outline_agent
    │
    ▼
[生成场景内容] ── LangGraph: scene_agent (per scene, 并行)
    │
    ▼
[生成教学动作] ── LangGraph: action_agent (per scene, 并行)
    │
    ├──(首个 Scene 就绪即跳转)──► [进入课堂 /classroom/:id]
    │                                │
    │                                ▼
    └──(剩余 Scene 后台推流)──────► [播放引擎]
                                     │
                                     ▼
                            按 action timeline 依次执行：
                             ├── SPEECH    (TTS + 字幕)
                             ├── SLIDE     (切换 PPT 页 / 动画)
                             ├── FOCUS     (高亮 slide 区域)
                             ├── WRITE     (白板绘制)
                             ├── INTERACT  (提问/测验/等待用户回答)
                             └── DISCUSS   (多 agent 对话)
```

> **渐进式进入课堂**：用户不等所有 Scene 生成完。只要 outline + 第 1 个 Scene 的 actions 就绪，就立刻跳转到课堂开始播放；剩余 Scene 由后台 SSE 持续推送到前端 `lessonStore`，侧边栏用生成状态徽章提示。目标：主题输入 → 进入课堂 **< 60 秒**。

## 5. 生成阶段的可见状态

每个生成阶段必须提供：

- 阶段名称、一句话说明
- 进度条或不确定动画
- "查看详情"可展开 agent trace（原始 LLM 调用、中间产物）
- 失败时明确错误原因与重试按钮

## 6. 导出能力（阶段 3 才做）

- **PPTX 导出**：每个 slide 一页，备注区写讲稿。
- **单文件 HTML 导出**：自包含，可离线播放；资源内联。

## 7. 国际化

- 默认支持 `zh-CN` / `en-US`。
- UI 文案走 `next-intl`。
- 生成的课程内容按用户当前语言生成；可一键切换重新生成译文。

## 8. 主题

- 支持 light / dark，跟随系统或手动切换。
- 所有颜色走 CSS variables + Tailwind theme。

## 9. 非功能需求

| 维度   | 要求                                                                                                        |
| ------ | ----------------------------------------------------------------------------------------------------------- |
| 性能   | 首页 TTI < 2s；课堂切场景 < 300ms                                                                           |
| 可用性 | 生成失败必须能回到上一步而不丢输入                                                                          |
| 安全   | API Key 存浏览器 **Dexie (IndexedDB) 加密存储**（阶段 1-2，禁止 localStorage）/ 后端 KMS 加密存储（阶段 4） |
| 可观测 | 每次 LLM 调用都要有 trace（model, tokens, latency, cost）                                                   |
| 可测试 | 核心逻辑（播放引擎、数据转换）必须有单测                                                                    |

## 10. 阶段外（Out of Scope for MVP）

- 用户系统 / 团队 / 权限 → 阶段 4
- 服务端持久化 → 阶段 3
- 协作编辑 → 不在本产品范围
- 真人直播 / 连麦 → 不做
